# Задание-спецификация: Инструмент обследования ITSM-процессов

## 1. Обзор задачи

Разработать desktop-приложение (Python + SQLite) для интерактивного проведения обследования ITSM-процессов по опроснику на основе ГОСТ 5492-3. Инструмент обеспечивает пошаговый обход вопросов, фиксацию ответов с временными метками, редактирование, сохранение и экспорт данных для последующей обработки.

## 3. Функциональные требования

### FR-01: Загрузка опросника
- При первом запуске импортировать `oprosnik.csv` в SQLite-базу
- Если данные уже импортированы — использовать существующие
- Поддержка кодировки CP1251

### FR-02: Навигация по вопросам
- Навигация по опроснику: вперёд / назад / к конкретному вопросу
- Фильтрация по процессу (INC, PRB и т.д.)
- Фильтрация: только обязательные / все вопросы
- Счётчик прогресса: "X из Y отвечено" (общий и по процессу)
- Визуальное выделение: отвечен / не отвечен / помечен для ревью

### FR-03: Ввод и редактирование ответов
- Поле для ввода текстового ответа (многострочный textarea)
- Поле оценки/статуса: выпадающий список (Соответствует / Частично / Не соответствует / Н/П — не применимо / Не проверено)
- Поле комментария
- Возможность прикрепить оценку соответствия по шкале 1–5
- Кнопка "Пометить для ревью" (флаг)
- Дата и время ответа фиксируются автоматически при сохранении
- Дата и время последнего изменения фиксируются при каждом обновлении

### FR-04: Сохранение
- Автосохранение при переходе к следующему вопросу
- Кнопка "Сохранить" (Ctrl+S)
- Поддержка нескольких сессий обследования (session_id)
- Возможность сохранить промежуточный результат и продолжить позже

### FR-05: Просмотр и редактирование ответов
- Табличный просмотр всех ответов текущей сессии
- Фильтрация по процессу, статусу, флагу ревью
- Сортировка по дате ответа, процессу, статусу
- Двойной клик / кнопка "Редактировать" — переход к форме вопроса
- Экспорт таблицы ответов в CSV

### FR-06: Управление сессиями
- Создать новую сессию (с именем/датой)
- Открыть существующую сессию
- Отобразить список сессий с прогрессом каждой
- Нельзя удалить сессию (только архивировать) — для сохранности данных

### FR-07: Отображение контекста вопроса
- Показывать подсказку для ответа (`answer_hint`) по требованию
- Показывать ожидаемые свидетельства (`expected_evidence`) по требованию
- Показывать метаданные: ГОСТ-код, приоритет, обязательность, процесс

## 5. Схема базы данных SQLite

### Таблица `questions`
```sql
CREATE TABLE questions (
    id              TEXT PRIMARY KEY,   -- DTN-ITSM-INC-01
    gost_code       TEXT,
    title_short     TEXT,
    description     TEXT,
    process_group   TEXT,
    standard_ref    TEXT,
    priority        TEXT,
    weight          TEXT,
    mandatory       TEXT,               -- 'обязательный' / 'дополнительный'
    management_area TEXT,
    process_name    TEXT,
    process_code    TEXT,               -- 'INC', 'PRB', etc. (извлекается из id)
    question_number INTEGER,
    question_text   TEXT,
    answer_hint     TEXT,
    expected_evidence TEXT
);
```

### Таблица `sessions`
```sql
CREATE TABLE sessions (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    name        TEXT NOT NULL,
    description TEXT,
    created_at  TEXT NOT NULL,          -- ISO 8601 UTC
    updated_at  TEXT NOT NULL,
    status      TEXT DEFAULT 'active'   -- 'active' / 'archived'
);
```

### Таблица `answers`
```sql
CREATE TABLE answers (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id      INTEGER NOT NULL REFERENCES sessions(id),
    question_id     TEXT NOT NULL REFERENCES questions(id),
    answer_text     TEXT,               -- свободный текст ответа
    compliance      TEXT,               -- 'compliant'/'partial'/'non_compliant'/'na'/'not_checked'
    score           INTEGER,            -- оценка 1-5, NULL если не выставлена
    comment         TEXT,
    review_flag     INTEGER DEFAULT 0,  -- 0/1 (помечен для ревью)
    answered_at     TEXT,               -- ISO 8601 UTC (первый ввод ответа)
    updated_at      TEXT NOT NULL,      -- ISO 8601 UTC (последнее изменение)
    UNIQUE(session_id, question_id)
);
```

### Таблица `app_state`
```sql
CREATE TABLE app_state (
    key     TEXT PRIMARY KEY,
    value   TEXT
);
-- Хранит: last_session_id, last_question_id, window_geometry и т.п.
```

## 8. Критерии приёмки

| ID | Критерий |
|----|---------|
| AC-01 | При первом запуске все 383 вопроса импортируются без ошибок |
| AC-02 | Ответ сохраняется в БД с корректными `answered_at` и `updated_at` |
| AC-03 | При повторном открытии сессии восстанавливаются все ранее введённые ответы |
| AC-04 | Фильтрация по процессу корректно ограничивает список вопросов |
| AC-05 | Экспорт CSV содержит все колонки таблицы `answers` + метаданные вопроса |
| AC-06 | Временные метки в БД хранятся в UTC; в UI отображаются в локальном времени |
| AC-07 | Приложение запускается командой `python main.py` без дополнительных настроек |
| AC-08 | БД открывается в DB Browser for SQLite и содержит корректные данные |
