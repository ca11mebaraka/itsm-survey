# HLD: ITSM Survey Tool — High-Level Design

## 1. Контекст системы

```
┌─────────────────────────────────────────────────────────────┐
│                      Рабочий ноутбук / ПК                   │
│                                                             │
│  ┌─────────────────────────────────────────────┐            │
│  │          ITSM Survey Tool (Python)          │            │
│  │                                             │            │
│  │   ┌───────────┐     ┌───────────────────┐   │            │
│  │   │  GUI App  │────▶│  SQLite Database  │   │  ◀── нет  │
│  │   │  (PyQt6)  │     │  survey_data.db   │   │   сети    │
│  │   └───────────┘     └───────────────────┘   │            │
│  │                              │               │            │
│  └──────────────────────────────┼───────────────┘            │
│                                 │                            │
│          ┌──────────────────────┼──────────────────┐         │
│          │                     │                  │         │
│     oprosnik.csv          export.csv         report tool    │
│   (импорт 1 раз)        (по запросу)        (pandas/BI)     │
└─────────────────────────────────────────────────────────────┘
```

**Ключевые решения:**
- Полностью offline — нет зависимостей от сети
- Один файл БД = всё состояние системы (легко бэкапить)
- Стандартный SQL-совместимый SQLite — прямой доступ к данным без API

## 4. Диаграмма последовательности: сохранение ответа

```
QuestionForm    AnswerService      Database
     │               │                │
     │──save_click───▶│               │
     │               │──get_answer(session_id, q_id)
     │               │                │──SELECT * FROM answers WHERE ...
     │               │                │──→ existing_answer or None
     │               │                │
     │               │ (если None) answered_at = utcnow()
     │               │ (иначе)     answered_at = existing.answered_at
     │               │ updated_at = utcnow()
     │               │                │
     │               │──INSERT OR REPLACE INTO answers (...)
     │               │                │──→ OK
     │               │                │
     │               │──UPDATE sessions SET updated_at = now() WHERE id = ?
     │               │                │
     │◀──answer_saved─│               │
     │               │                │
     │ обновить статус вопроса в NavigationPanel
```

## 7. Варианты UI-фреймворка

| Критерий | Tkinter | PyQt6 |
|----------|---------|-------|
| Установка | Встроен в Python | `pip install PyQt6` |
| Внешний вид | Нативный (устаревший) | Современный, кастомизируемый |
| Таблицы | ttk.Treeview (ограничен) | QTableView (полноценный) |
| Многострочный текст | Text widget | QTextEdit |
| Сложность | Низкая | Средняя |
| **Рекомендация** | MVP/прототип | **Финальная версия** |

**Рекомендация: PyQt6.** Таблицы и форма ввода с большим количеством полей значительно удобнее реализуются на PyQt6. Размер зависимости ~50 МБ, устанавливается штатным `pip`.

## 8. Ключевые технические решения

| # | Решение | Обоснование |
|---|---------|-------------|
| 1 | SQLite без ORM | Прямой доступ к данным через pandas/DB Browser без лишних абстракций |
| 2 | `INSERT OR REPLACE` для ответов | Идемпотентное сохранение без проверки на существование |
| 3 | UTC в БД + конвертация в UI | Корректная работа при смене часового пояса / переносе БД |
| 4 | process_code как денормализованное поле | Быстрая фильтрация без парсинга ID при каждом запросе |
| 5 | Автосохранение при переходе | Защита от потери данных при непредвиденном закрытии |
| 6 | Один файл БД | Простота бэкапа: `cp survey_data.db survey_data_backup_20250331.db` |
| 7 | Миграционная схема БД | Возможность добавлять поля без пересоздания БД |

## 9. Риски и ограничения

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| Повреждение CSV при изменении кодировки | Средняя | Валидация при импорте + логирование пропущенных строк |
| Одновременная работа двух экземпляров приложения | Низкая | SQLite WAL-mode включён по умолчанию |
| Большой объём данных (будущее расширение) | Низкая | Индексы на `session_id`, `process_code` — достаточно для 10K+ ответов |
| Потеря несохранённых данных | Средняя | Автосохранение при навигации + подтверждение при закрытии |
