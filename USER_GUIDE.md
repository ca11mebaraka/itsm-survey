# Руководство пользователя — ITSM Survey Tool

## Кто использует этот инструмент

**Аналитик** или **обследователь**, который проводит оценку зрелости ITSM-процессов в организации. Вы задаёте вопросы представителям заказчика, фиксируете ответы и сохраняете результаты для последующего анализа.

---

## Раздел 1. Начало работы

### 1.1 Открыть инструмент

Перейдите по ссылке: **https://ca11mebaraka.github.io/itsm-survey/**

Откроется стартовый экран:

```
┌─────────────────────────────────────────┐
│           ITSM Survey Tool              │
│                                         │
│  [ ✨ Начать новое обследование ]       │
│                                         │
│             — или —                     │
│                                         │
│  [ 📂 Продолжить (загрузить файл .db) ] │
│                                         │
│  💾 Все данные хранятся в файле .db     │
│     на вашем компьютере                 │
└─────────────────────────────────────────┘
```

### 1.2 Первый запуск — создать новую сессию

1. Нажмите **«✨ Начать новое обследование»**
2. Введите название сессии, например: `ООО Ромашка — март 2026`
3. При желании добавьте описание: `Первичное обследование`
4. Нажмите **«Создать»**

> **Совет:** Называйте сессию так, чтобы было понятно, какая это организация и дата обследования.

### 1.3 Продолжить прерванное обследование

1. Нажмите **«📂 Продолжить (загрузить файл .db)»**
2. Найдите на компьютере ранее сохранённый файл — он называется примерно `itsm_survey_ООО_Ромашка.db`
3. Нажмите **«Открыть»** в диалоге выбора файла
4. Инструмент загрузит все ваши предыдущие ответы и откроет сессию

После открытия сессии вы видите три зоны:

```
┌──────────────────────────────────────────────────────────────┐
│ ITSM Survey │ Название сессии │ 5/383 (1%) │ [💾] [📋] [⬇] │  ← Шапка
├───────────┬──────────────────────────────────────────────────┤
│           │                                                  │
│  Список   │    Форма вопроса                                 │
│ процессов │                                                  │
│ с прогрес-│    [Вопрос] [Подсказка ▼] [Свидетельства ▼]     │
│   сом     │    [Поле ответа]                                 │
│           │    [Статус] [Оценка] [Комментарий]              │
│           │    [⚐ Для ревью]                                │
├───────────┤──────────────────────────────────────────────────┤
│           │    [◀ Назад]    Вопрос 5 из 383    [Вперёд ▶]  │  ← Навигация
└───────────┴──────────────────────────────────────────────────┘
```

### Шапка (верхняя полоса)
| Элемент | Назначение |
|---------|-----------|
| `💾 Сохранить файл` | Скачать файл `.db` на компьютер |
| `📋 Все ответы` | Таблица всех введённых ответов |
| `⬇ CSV` | Экспортировать ответы в Excel-формат |
| `🗂 Сессия` | Переключиться между сессиями |
| Счётчик `5/383 (1%)` | Глобальный прогресс: отвечено / всего |

### Левая панель (процессы)
- Список 15 ITSM-процессов с индикаторами прогресса
- Нажмите на процесс → переход к вопросам этого процесса
- Полоска становится **зелёной**, когда все вопросы процесса заполнены
- **Фильтры** над списком:
  - «Только обязательные» — скрыть дополнительные вопросы
  - «Без ответа» — показать только незаполненные
  - «На ревью» — вопросы, помеченные для повторной проверки

## Раздел 3. Ответ на вопросы

### 3.1 Читать вопрос

Каждый вопрос содержит:
- **Текст вопроса** — главное, что нужно задать собеседнику
- **Подсказка для ответа** *(нажмите «💡 Подсказка»)* — что именно нужно выяснить, примеры
- **Ожидаемые свидетельства** *(нажмите «📎 Свидетельства»)* — какие документы/факты подтвердят ответ

> **Рекомендация:** Перед интервью прочитайте подсказку — она помогает задать уточняющие вопросы.

### 3.2 Заполнить ответ

**Поле «Текст ответа»** — свободный текст, пишите всё что сказал собеседник.

**Статус соответствия** — выберите один из вариантов:

| Статус | Когда выбирать |
|--------|---------------|
| 🟢 **Соответствует** | Требование полностью выполнено, есть подтверждение |
| 🟡 **Частично** | Требование выполнено частично или только формально |
| 🔴 **Не соответствует** | Требование не выполнено |
| 🟣 **Не применимо** | Процесс/требование не актуально для данной организации |
| ⚪ **Не проверено** | Ответ ещё не получен (по умолчанию) |

**Оценка ★★★☆☆** — дополнительная числовая оценка от 1 до 5 звёзд.
Необязательна, используйте по усмотрению методологии.

**Комментарий** — ваши личные заметки, наблюдения, источники.

**⚐ Пометить для ревью** — флаг «надо вернуться». Используйте если:
- Ответ нужно уточнить
- Получен противоречивый ответ
- Нужно запросить дополнительные документы

### 3.3 Навигация

| Действие | Как |
|----------|-----|
| Следующий вопрос | Кнопка **«Вперёд ▶»** или `Alt + →` |
| Предыдущий вопрос | Кнопка **«◀ Назад»** или `Alt + ←` |
| Перейти к процессу | Нажать на процесс в левой панели |
| Сохранить и скачать файл | Кнопка **«💾»** или `Ctrl + S` |

> **Важно:** Ответы сохраняются автоматически при переходе к следующему вопросу.
> Но файл `.db` нужно скачивать вручную — иначе данные будут утеряны при закрытии вкладки.

## Раздел 4. Сохранение данных

### ⚠️ Это самое важное!

Инструмент хранит данные **только в памяти браузера**. Если закрыть вкладку без сохранения — все несохранённые ответы будут утеряны.

### Когда сохранять

Следуйте простому правилу: **«Ответил на 10 вопросов — скачай файл»**.

Подсказки в интерфейсе:
- Кнопка `💾` становится **оранжевой** и пульсирует при наличии несохранённых ответов
- В шапке появляется счётчик «💾 Не сохранено: 5»

### Как сохранить

1. Нажмите кнопку **«💾 Сохранить файл»** (или `Ctrl + S`)
2. Файл `itsm_survey_НазваниеСессии.db` скачается в папку «Загрузки»
3. Кнопка становится зелёной — всё сохранено

### Где хранить файл

Рекомендации:
- Создайте папку для проекта: `Обследование / ООО Ромашка / Данные /`
- Храните все версии файла (не перезаписывайте)
- Называйте файл с датой: `itsm_survey_Ромашка_2026-03-31.db`
- Создайте резервную копию на флешке или в облаке

## Раздел 5. Просмотр и правка ответов

### Таблица всех ответов

Нажмите **«📋 Все ответы»** в шапке.
Откроется таблица со всеми заполненными вопросами:

| Столбец | Описание |
|---------|----------|
| Процесс | Код и номер вопроса |
| Вопрос | Текст вопроса (при наведении — полный текст) |
| Статус | Цветной значок соответствия |
| Оценка | Звёзды |
| ⚑ | Флаг ревью |
| Обновлён | Дата последнего изменения |

Чтобы отредактировать ответ — **нажмите на строку** в таблице.
Вы вернётесь к форме вопроса и сможете изменить ответ.

### Фильтры для ревью

Используйте фильтры в левой панели:
- «На ревью» → просмотреть все помеченные вопросы
- «Без ответа» → быстро найти незаполненные вопросы

## Раздел 6. Экспорт в Excel

По завершении обследования:
1. Нажмите **«⬇ CSV»** в шапке
2. Скачается файл `survey_НазваниеСессии.csv`
3. Откройте в Excel — файл автоматически открывается с правильной кодировкой

Файл CSV содержит все данные: вопросы, ответы, статусы, оценки, даты, ссылки на ГОСТ.

## Раздел 7. Несколько сессий

Один файл `.db` может содержать несколько сессий (например, разные периоды или организации).

Чтобы создать новую сессию в том же файле:
1. Нажмите **«🗂 Сессия»** в шапке
2. Нажмите **«+ Новая сессия»**
3. Введите название и нажмите **«Создать»**

Переключение между сессиями:
1. Нажмите **«🗂 Сессия»**
2. Выберите нужную сессию из списка

## Раздел 9. Частые вопросы

**Q: Закрыл вкладку случайно. Данные пропали?**
A: Если вы скачивали файл `.db` — загрузите его через «Продолжить». Если нет — к сожалению, несохранённые ответы не восстановить. Сохраняйте чаще.

**Q: Можно ли работать офлайн?**
A: Нет. Для первой загрузки страницы нужен интернет. После загрузки можно работать без сети, но файл `.db` нельзя будет скачать до восстановления соединения.

**Q: Файл .db — что это такое?**
A: Стандартный формат базы данных SQLite. Открывается программой [DB Browser for SQLite](https://sqlitebrowser.org/) и читается через Python/pandas.

**Q: Потерял файл .db — как восстановить?**
A: Проверьте папку «Загрузки» — браузер сохраняет все скачанные файлы туда. Также проверьте облачное хранилище, если оно синхронизирует папку «Загрузки».

**Q: Можно работать одновременно с коллегой?**
A: Нет, инструмент не поддерживает одновременную работу нескольких пользователей. Рекомендуется разделить вопросы по процессам и работать по очереди, затем объединить данные (см. руководство администратора).

**Q: Можно скопировать данные из одного файла .db в другой?**
A: Да, это делает администратор с помощью DB Browser for SQLite или Python-скрипта.
